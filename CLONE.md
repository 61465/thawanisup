# 📦 دليل الاستنساخ السريع — عميل جديد في 30 دقيقة

هذا هو الـ playbook الذي تتبعه كل مرة يدفع لك فيها عميل جديد لإطلاق بوت متجره.

---

## 🎯 الهدف

في نهاية هذه الـ 30 دقيقة:
- بوت يعمل على رقم واتساب العميل
- Google Sheet خاص بمنتجاته
- لوحة Railway منفصلة لكل عميل (لاحتساب الفواتير + العزل)
- العميل قادر يعدّل أسعاره من هاتفه بدون مساعدتك

---

## ✅ Checklist قبل البدء

اطلب من العميل قبل الموعد:
- [ ] رقم واتساب بزنس مخصص للمتجر (ليس رقمه الشخصي إن أمكن)
- [ ] قائمة منتجات (Excel/Word/كتابة) مع: اسم، فئة، سعر، وصف قصير
- [ ] لوجو المتجر (اختياري)
- [ ] حساب فيسبوك يستطيع به الدخول
- [ ] جواز/هوية للتحقق من Business Manager

## 🔁 خطوات الاستنساخ (10 خطوات)

### 1️⃣ Fork المستودع (دقيقة)
```bash
# على جهازك:
gh repo fork your-username/whatsapp-cafe-bot --clone --fork-name whatsapp-bot-CLIENT_NAME
cd whatsapp-bot-CLIENT_NAME
```

### 2️⃣ تخصيص اسم المتجر (دقيقة)
عدّل **فقط** القيم في `.env` لاحقاً — الكود نفسه لا يحتاج تعديل.

> إذا أراد العميل ألواناً/إيموجي مختلفة، عدّل في `src/server.js`:
> - دالة `sendCategoryMenu` — الإيموجيات
> - دالة `sendWelcome` — رسالة الترحيب

### 3️⃣ نسخ Google Sheet (3 دقائق)
1. افتح القالب الأصلي (الذي عملته للعميل الأول)
2. **File → Make a copy** → اسم: "منتجات [اسم متجر العميل]"
3. **Share** → العميل بصلاحية Editor (ليعدّل الأسعار)
4. **File → Share → Publish to web → CSV** → انسخ الرابط

### 4️⃣ Meta App للعميل (10 دقائق — العميل يجب أن يكون معك)
> اتبع [DEPLOYMENT.md](./DEPLOYMENT.md) خطوة 1.1 إلى 1.4
> 
> العميل يستخدم حسابه الشخصي، أنت ترشد.

✏️ احفظ:
- `WHATSAPP_TOKEN` (Permanent)
- `WHATSAPP_PHONE_ID`
- App ID
- Business Manager URL

### 5️⃣ Push كود العميل (دقيقة)
```bash
git remote set-url origin https://github.com/YOU/whatsapp-bot-CLIENT_NAME.git
git push -u origin main
```

### 6️⃣ Railway Project جديد (3 دقائق)
1. **railway.app** → **New Project** → **Deploy from GitHub** → اختر مستودع العميل
2. **Variables** — انسخ من جدولك المرجعي:
   ```
   WHATSAPP_TOKEN=...
   WHATSAPP_PHONE_ID=...
   VERIFY_TOKEN=client_name_secret_2026
   OWNER_PHONE=...
   STORE_NAME=...
   CURRENCY=ر.س
   DELIVERY_FEE=...
   SHEET_CSV_URL=...
   ```
3. **Generate Domain** → احفظ الرابط

### 7️⃣ ربط Webhook (3 دقائق)
> [DEPLOYMENT.md](./DEPLOYMENT.md) خطوة 4 بالضبط

### 8️⃣ اختبار end-to-end (5 دقائق)
1. أرسل "السلام عليكم" لرقم العميل من رقمك (المُسجّل في To list)
2. اختبر: قائمة → منتج → سلة → فاتورة → تأكيد
3. تحقق من وصول إشعار الطلب لرقم العميل
4. افتح `https://[client-domain].railway.app/orders?token=OWNER_PHONE`

### 9️⃣ تسليم العميل (3 دقائق)
أرسل للعميل:
- ✅ رابط Google Sheet مع تعليمات قصيرة لتعديل الأسعار
- ✅ رابط `/orders?token=...` لمراجعة الطلبات
- ✅ رقم الواتساب الذي يستقبل الإشعارات (نفس OWNER_PHONE)
- ✅ فيديو/سكرين-كاست قصير (1-2 دقيقة) لكيفية الاستخدام

### 🔟 App Review للإنتاج (يتم لاحقاً)
> العميل يكمل [DEPLOYMENT.md](./DEPLOYMENT.md) قسم "Anti-Test"
> 
> يستغرق 24-72 ساعة موافقة Meta. بعدها البوت يخدم أي رقم.

## 💰 نموذج التسعير المقترح

| الخدمة | السعر |
|--------|-------|
| إعداد البوت وتفعيله | 500-800 ر.س لمرة واحدة |
| اشتراك شهري (استضافة + صيانة + دعم) | 100-200 ر.س/شهر |
| تخصيص الشجرة (منتجات > 30، إضافة فئات) | 200-400 ر.س |
| ربط بوابة دفع (مدى/STC Pay) | 500-1000 ر.س |

> Railway مجاناً يكفي لمتجر صغير (~1000 طلب/شهر). للمتاجر النشطة استخدم خطة Hobby بـ $5/شهر.

## 🚨 ما يجب أن تتجنبه

1. **لا تشارك Token واحد بين عملاء** — كل عميل له System User منفصل
2. **لا تستخدم نفس VERIFY_TOKEN** — قابل لـ Token leak
3. **لا تربط أكثر من عميل بنفس مستودع GitHub** — أصعب في إدارة الفروق
4. **لا تنسَ Generate Domain في Railway** — webhook لن يعمل بدون دومين
5. **لا تنشر .env على GitHub أبداً** — موجود في `.gitignore`، لكن تحقق دائماً
